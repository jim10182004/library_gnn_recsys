# 口試展示流程 — 給「自己」的 SOP

> 教授坐下、學長姐圍觀、口試 15-20 分鐘簡報 + 5-10 分鐘 Q&A
> **這份是你的對嘴稿** — 照這順序點、照這劇本講

---

## 🔧 開場前 5 分鐘準備（口試開始前）

### Checklist（在自己的筆電上做）
- [ ] 開啟 `docs/圖書館GNN推薦系統_簡報.pptx` 進全螢幕
- [ ] 開啟一個 PowerShell（或終端機）放第 2 螢幕
- [ ] 在 PowerShell 跑：
  ```powershell

  uvicorn api.main:app --host 127.0.0.1 --port 8000
  ```

- [ ] 等到看到 `Application startup complete` + `[FastAPI] 模型就緒`（約 30 秒）

- [ ] 右上角點「📊 統計」確認 `best_model = lightgcn_multi_opt`
- [ ] **備案**：如果 demo 掛掉，PPT 第 18 頁有 demo 截圖可講

## 📋 簡報順序（22 頁，預計 12-15 分鐘）

> 每頁停留時間是建議值。**講得快不會扣分，講得慢可能會被打斷**。

## 🎬 Demo 詳細劇本（口試的高潮）

> 切到瀏覽器 `http://127.0.0.1:8000`，**講話節奏要慢、滑鼠移動要慢**，
> 教授才看得清楚你在幹嘛。

### 劇本 A：Persona 模式（穩定示範）

```
1. 「這是我為公開展示做的 Demo，沒有真實讀者 ID，只有 11 個人物原型。」
   ← 滑鼠指左側 11 個 persona 卡

2. 「我點『日系推理小說迷』」← 點 🔍 卡
   → 系統會顯示 4 本種子書（白金數據、嫌疑犯X、解憂雜貨店、死了七次的男人）
   → 並列出 10 本推薦書

3. 「注意推薦的第 1 本『主教殺人事件』，是英國推理小說 — 跟種子書類別相符」
   ← 滑鼠移到 #1 書

4. 「每本書下面有彩色標籤解釋『為什麼推薦這本』 — 可能是同作者、同類別、或相似讀者群也借」
   ← 滑鼠指 badge

5. 「分數 0.83 是 cosine similarity，越接近 1 表示模型越有信心」
   ← 滑鼠指 score
```

### 劇本 B：自訂模式（如果時間夠）

```
1. 「點上方『自訂喜歡的書』」← 切到搜尋頁

2. 在搜尋框輸入「東野」← 即時跳出書名 dropdown

3. 「我選 3 本東野圭吾的書」← 點 3 本加入

4. 「按推薦」→ 模型用我選的 3 本當 seed → 推薦類似 10 本

5. **核心點**：「同樣的模型，同樣的計算邏輯，沒有 hard-code 任何規則 — 純粹是學到的 embedding 在運作」
```

### 劇本 C：比較模式（最炫但最有風險）

```
1. 「點『比較兩個原型』」

2. 「左邊選『兒童英文書愛好者』，右邊選『日系推理小說迷』」

3. 推薦結果並排顯示，**重疊的書會被黃框框起來**

4. **關鍵句**：「兩個 persona 的推薦完全沒重疊（0 本黃框）— 證明模型能精準區分不同讀者群」
```

## 🤔 預期 Q&A（按可能性排序）

### Q1（90% 會問）：「為什麼選 LightGCN？」
**A**：「LightGCN 是 2020 SIGIR 的 baseline 共識，原作者證明簡化版勝過 NGCF 複雜版。我在自己的資料上重現了這個結論：LightGCN R@10=0.265 > NGCF 0.264。」

### Q2（80% 會問）：「資料怎麼來的？有隱私問題嗎？」
**A**：「來自一間市立圖書館的內部資料，已經完全去識別化 — 讀者 ID 是匿名編號，沒有姓名、地址。論文與所有 Demo 都不顯示真實讀者 ID。」

### Q3（70% 會問）：「為什麼不用 ChatGPT？」
**A**：（用我的對照表回答）「成本（每月 $300 vs 0）、隱私（傳給 OpenAI 有風險）、速度（1-3 秒 vs 17 ms）、個人化（ChatGPT 不知道讀者個別模式）」。詳見白話版第 376 行。

### Q4（60% 會問）：「Coverage 是什麼？為什麼重要？」
**A**：「Coverage = 推薦的書佔全館的比例。Popular 只推前 30 本（Cov=0.001），我的 Optuna 模型推到全館 26.5% — 對長尾書曝光是巨大進步。」

### Q5（50% 會問）：「能上線嗎？」
**A**：「技術上可以 — GPU 推薦延遲 17 ms，遠超即時需求。但實際部署需要：(1) 線上 A/B 測試（半年）、(2) ILS 整合（1-2 年）、(3) IRB 倫理審查。論文 5.4 節有詳細路徑。」

### Q6（40% 會問）：「為什麼 BERT 沒大幅提升？」
**A**：「我 2 個假設：(1) 借閱資料的協同訊號已經很強，書名語意是冗餘訊號；(2) BERT 用的是通用多語模型（multilingual MiniLM），不是針對中文書名 fine-tune。改用 BERT-wwm-ext 應該會更好。」

### Q7（30% 會問）：「樣本數 n=3 seeds 太少？」
**A**：「同意樣本小，但 std 只有 0.0005，是模型差距 (0.005) 的 1/10，paired t-test p<0.005 仍顯著。論文 4.4 節有完整檢定。」

### Q8（30% 會問）：「為什麼書封 CNN 失敗？」
**A**：「Open Library 中文書封覆蓋率僅 4.4%，所以 1500 本只成功 66 本，feature 主要是零向量無法監督學習。誠實寫進論文限制章節 5.3，未來改用 Google Books API 應可達 70%+。」

### Q9（20% 會問）：「冷啟動讀者反而效果好？」
**A**：「對，這是反直覺結果但合理 — 冷啟動讀者通常借熱門書，跟 Popular 推薦重疊；重度讀者借閱多元，個人化的難度才真正展現。論文 4.2.5 節。」

### Q10（陷阱題）：「你的指標是線下評估，能保證上線一定好嗎？」
**A**：「不能。離線評估的限制是只能看『讀者下個月實際借了什麼』，但讀者真正想借的書可能因為館藏沒有、時間不對而沒借到。所以未來工作明確列出『線上 A/B 測試是商品化前必要工作』。」

### 應對「不會的題目」
- **千萬不要硬掰** → 「這部分我目前沒有完整答案，但我的猜測是 ___，下一步會 ___」
- **承認 + 給方向**比「胡說一通」好太多

## ⚠️ 危險區（這些絕對不要說）

| ❌ 不要講 | ✅ 改說 |
|---|---|
| 「ChatGPT 比我厲害」 | 「ChatGPT 適合不同情境，本研究在成本/隱私/速度都更優」 |
| 「我覺得這個沒用」 | 「這個結果不如預期，但我學到了 ___」 |
| 「這部分我抄論文的」 | 「這部分參考 LightGCN 原論文 (He 2020)，我的貢獻在 ___」 |
| 「我亂試的」 | 「我用 Optuna 系統性搜尋了 20 組超參數」 |
| 「應該吧 / 大概 / 我不確定」 | 「目前的觀察是 ___，但需要更多實驗驗證」 |
