How to use

怎麼玩這個 demo？

點開下面任一個 mode 看詳細說明，或直接跳到 demo 開始試。

🎭

11 種人物原型

最簡單的玩法

點任一個 emoji 卡（如「日系推理迷 🔍」），系統立刻給推薦。 不需要任何輸入。

🔍 內部怎麼運作？（白話版）

1. 每個原型背後**藏了 4 本代表書**（像「日系推理迷」= 白金數據+嫌疑犯X+解憂雜貨店+死了七次的男人）

2. 模型從 130 萬筆借閱學到「**每本書都有一張 64 個數字組成的『興趣指紋』**」

3. 把這 4 本的指紋**取平均** → 得到一張「假裝是這種讀者」的指紋

4. 拿這張平均指紋跟**全館 3 萬本書**比對，找最像的 10 本（不會推薦原本那 4 本）

→ 比喻：4 本書 = 你朋友的興趣標籤；模型 = 紅娘；推薦結果 = 紅娘介紹的對象

✏️

新讀者引導 (Cold-start)

3 步驟個人化

搜尋你看過、喜歡的 3-5 本書 → 加入 → 按「產生推薦」。

⚙️ 為什麼要選 3-5 本？

• 1-2 本：訊號太弱，virtual user vector 容易偏

• 3-5 本：剛好——能反映你的口味又不失多樣

• 5+ 本：每多一本邊際遞減，但會更穩定

推薦結果中按「+ 我也借過」可即時加入並重算

🆚

比較兩個原型

展示模型區分能力

左右各選一個 persona → 並排顯示推薦書單。 重疊的書會被 黃框 標出。

⚙️ 觀察重點

• 「兒童英文 vs 日系推理」→ 預期 0 重疊，證明模型能精準區分

• 「程式設計師 vs 學術派」→ 預期 2-3 本重疊，反映兩群人都讀數學/演算法

• 重疊書數越接近 0 = 模型區分力越強；越接近 K = 兩 persona 興趣重疊

「嚴格同類別」Toggle

推薦書單標題旁的開關，控制是否強制同類別優先：

關閉（預設）：純 LightGCN 行為訊號 — 可能跨類別 （例：程式書 → 月老）。展示 GNN 能找隱藏興趣。
開啟：同類別 +0.15 分、異類 -0.05 分 → 全部優先同類書（例：程式書 → 全部科學/應科）。

About

技術細節

本系統如何工作？

📚 資料是哪來的？

某間市立圖書館 2025 全年的 真實借閱紀錄（130 萬筆借閱 + 32 萬筆預約）。讀者 ID 已替換成匿名編號，沒有姓名、聯絡資訊。

🧠 模型在做什麼？

把每位讀者跟他借過的書 連線，畫成一張超大的網。模型學「跟你借過類似書的人，他們還借了什麼」 — 跟 Netflix / YouTube 同一套邏輯，只是換成書。

🎯 推得有多準？

用 12 月真實借閱資料測試（模型沒看過）：每 10 本推薦平均 2.7 本是讀者真的會借的。43% 的讀者至少 1 本命中。比「推熱門書」聰明，覆蓋的書多 265 倍。

隱私聲明

資料來源已去識別化，本 demo 不直接顯示任何真實讀者 ID。Demo 中的「人物原型」皆為合成範例，並透過模型計算合成讀者向量產生推薦。

推薦怎麼算出來？

  1. 1 你選書（點 persona 或自訂 3-5 本喜歡的書）
  2. 2 取這幾本書的「興趣指紋」平均 → 得到一張「假裝是這種讀者」的指紋
  3. 3 跟全館 29,685 本書比指紋，找最像的 Top 10

💡「指紋」= 64 個浮點數，模型訓練時自動學出來的 — 完全沒人為定義， 反映「跟誰一起被借」的潛在規律。

↓ 同件事用程式碼寫只要 3 行
# 合成讀者向量 → 找最像的書
user_vec = item_embs[liked].mean(dim=0)
scores = cosine(user_vec, all_items)
top_k = scores.argsort()[-k:]
