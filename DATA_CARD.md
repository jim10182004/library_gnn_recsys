## 切分方式

### 時序切分（不是隨機切！）

```
train = data[date < '2025-11-01']  # 1-10 月
val   = data[(date >= '2025-11-01') & (date < '2025-12-01')]  # 11 月
test  = data[date >= '2025-12-01']  # 12 月
```

**重要**：val/test 中的 `(user, item)` 對若已在 train 出現，會從 val/test 移除（避免 leakage）。

### K-core ≥ 5 過濾

迭代執行直到收斂：
1. 移除互動 < 5 的讀者
2. 移除被借 < 5 次的書
3. 重新計算
4. 重複直到所有節點都 ≥ 5

收斂後規模見上「處理後」表。

## 資料前處理（src/preprocess.py）

### 髒資料處理

| 欄位 | 問題 | 處理方式 |
|---|---|---|
| `birth_year` | 字串 "20uu"、"19  "、"19gg" | 強制轉 int，失敗則 NaN |
| `gender` | "男"/"女"/"M"/"F"/"未知"/"其他" 不一致 | 統一映射為 ["男", "女", "其他"] |
| `book_isbn` | 含空白、"-"、空字串 | strip + remove dashes |
| `borrow_date` | "2025/3/5"、"2025-3-5"、"2025年3月5日" | pd.to_datetime 各種格式 |
| `book_category` | "8 語文文學"、"8語文文學"、"8" | 取第一個數字字 [0-9] |
| `user_id` | 偶爾重複（系統 bug） | 去重 |

### 性別重新清洗
原始有 100+ 種寫法（含 "未填"、"unknown"、空字串等），重新映射為三類。
詳見 `src/reclean_gender.py`。

## 隱私處理

### 已採取的措施
1. **讀者 ID 匿名化**：圖書館給的就是匿名編號（如 `A0123456`），無姓名/地址/聯絡資訊
2. **完全去除個人識別資訊**：原始檔案就無 PII，本研究不接觸
3. **demo 不顯示真實 ID**：公開版（`app_public.py` / FastAPI）只接受 4 本書 seed
4. **論文/PPT 不揭露機構**：統一稱「某市立圖書館」
5. **環境變數隱藏路徑**：`LIBRARY_RAW_DIR` 透過 `.env` 設定，不進 git

### 風險評估
- **再識別風險**：低，因無 PII 可交叉比對
- **群體再識別**：理論上若已知某讀者的 5+ 本借閱書清單，可能識別其匿名 ID。為此，公開 demo 不允許上傳真實借閱史
- **書本敏感性**：少數書名可能涉及醫療/政治/性向等敏感主題；本系統不顯示「他借過 X」這類描述，僅推薦未借過的書

## 已知偏差 (Bias)

### 人口統計
- 性別：女性占略多數
- 年齡：35-49 歲為最大族群（ML/技術書、職場書最多借閱）
- 兒童（< 18）：借閱集中在繪本與童書系列

### 內容偏差
- 中圖類別 8（語文文學）佔 35% 的借閱
- 工程類書籍（5xx-6xx）借閱量遠少於文學/藝術
- 中文書佔 95%+，英文書多為兒童繪本與技術書

### 時間偏差
- 暑期借閱量明顯高峰
- 月底借閱量比月初少（還書期影響）

## 限制 (Limitations)

1. **無評分資料**（隱式回饋）：只能假設「借過」=「感興趣」
2. **無「未借但看過」資料**：不知道讀者實際在書架前停下卻沒借
3. **無線上瀏覽資料**：缺 OPAC 點擊或推薦面板互動資料
4. **單一圖書館**：跨圖書館遷移效果未驗證
5. **2025 年 12 個月**：未驗證跨年趨勢
6. **未涵蓋電子書**：本資料僅實體書借閱

## 倫理使用聲明

本資料**僅用於本畢業專題的學術研究**。下列用途**禁止**：
- 商業推薦系統的訓練資料
- 重新識別任何讀者
- 推銷、廣告或第三方分析
- 公開散布原始資料或處理後 parquet

如需重現實驗：請依照 [README.md](README.md) 步驟，從你自己的圖書館資料開始。

## 取得資料

本專案不公開原始資料或 processed parquet（受授權限制）。

如需相同 pipeline 在你的圖書館資料上跑：
1. 取得你機構的借閱資料
2. 設定 `.env` 中的 `LIBRARY_RAW_DIR`
3. 執行 `python src/preprocess.py`
4. 後續流程相同
